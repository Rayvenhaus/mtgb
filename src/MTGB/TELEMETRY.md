## The Short Version

MTGB can optionally send anonymous usage statistics
to help improve the software for the community.

- **It is opt-in. Default off. Always.**
- **No personal data is ever collected.**
- **No IP addresses. No names. No credentials.**
- **You can opt out at any time. Immediately. No forms.**

That's the whole story. Everything below is the
long version for those who want it.

---

## What Is Collected

When you enable anonymous telemetry, MTGB sends
one ping per day containing the following:

### Installation identity
- A randomly generated anonymous install ID
  — a GUID created on first run, stored locally,
  never tied to a person, account or organisation
- MTGB version number
- Windows version and build number

### Usage data
- Number of printers connected — count only, no names
- Printer integration type — "Bambu", "Prusa" etc
- Printer model brand and name — e.g. "Bambu Lab", "X1C"
- Which notification event types are enabled
  — a list of event IDs, no content
- Whether grouping is enabled
- Whether webhooks are enabled
- Whether quiet hours are enabled
- Whether sound is enabled
- Poll success and failure counts since last ping
- Toast delivery success and failure counts since last ping

### That's it

There is nothing else. The scribes have been
instructed accordingly.

## What Is Never Collected

The following are never collected, never transmitted,
never stored, and never will be:

- API keys or authentication tokens
- SimplyPrint organisation IDs
- Printer names
- Job filenames or print content
- Job history or notification content
- IP addresses
- Location data beyond state/territory level
  if you have opted into the community map
  (see below)
- Anything that could identify a person,
  business or organisation

## The Community Map

Separately from telemetry, MTGB offers an optional
community map showing where in the world MTGB is
being used.

This is a completely separate opt-in from telemetry.
You can enable telemetry without the map, the map
without telemetry, both, or neither.

### What the map records
- Your country — e.g. "Australia"
- Your state or territory if applicable
  — e.g. "Victoria"
- Country only for countries without a
  state/territory system — e.g. "Fiji"
- Your anonymous install ID — to allow opt-out

### What the map never records
- City, suburb, postcode or street
- Coordinates more precise than
  state/territory level
- Anything identifying

### Resolution policy
The minimum granularity is state or territory level
for countries that have them. For all other countries
it is country level only. This resolution is fixed
and will never be made more precise.

### Opting out
Settings → Privacy → Community map → Remove my dot.

Your registration is deleted immediately from the
database. The Ministry will file the departure form
without complaint.

## Where the Data Goes

Telemetry and community map data is sent to:
community.myndworx.com

This server is operated by the MTGB project maintainer
(Myndworx Asylum). The data is stored in a MariaDB database
on that server. It is not shared with any third party.
It is not sold. It is not used for advertising.
It is used to understand how MTGB is being used
so it can be improved.

The server is located in Sydney, AU. If you are subject 
to GDPR or similar privacy legislation, note that 
state/territory level location data is generally not 
considered personal data at this resolution. If you have 
concerns, simply do not enable telemetry or the community map.

## How to Opt Out

### Telemetry
Settings → Privacy → Anonymous telemetry → toggle off.

Takes effect immediately. The next scheduled ping
will not fire. No confirmation required. No forms.

### Community map
Settings → Privacy → Community map → Remove my dot.

Your record is deleted from the database immediately.
The status indicator will update to confirm removal.

### Both at once
If you want to disable everything in one action,
Settings → Privacy → turn off the telemetry toggle
and click Remove my dot if registered.

## During the Induction

The Induction wizard presents telemetry and the
community map as two separate opt-in choices,
both defaulting to OFF.

If you exit the Induction without completing it,
no data is ever sent. Any partial selections are
discarded. The install is treated as if the
Induction never happened.

## The Install ID

The anonymous install ID is a randomly generated
GUID — a 128-bit number that looks like this:
12345678-1234-4234-b234-123456789abc

It is generated by your computer using a
cryptographically secure random number generator.
It has no relationship to your Windows user account,
your SimplyPrint account, your hardware, your IP
address, or anything else. It is a number that
was rolled at random and stored locally.

It exists solely to allow the server to recognise
when a daily ping comes from a previously seen
installation — so it counts installs, not pings —
and to allow you to remove your community map
registration if you choose.

MTGB is open source under the MIT licence.

There are no hidden fields. There are no secondary
data streams. What you see in the code is what runs.
